## Overview

A day-in-the-life walkthrough of an L1 SOC analyst. This room establishes what the role involves in practice: the tools in use, the types of alerts handled, the decision-making process during triage, and how the analyst fits into the broader SOC workflow.

---

## The L1 Analyst Role

The L1 analyst is the SOC's first responder. Every alert generated by the SIEM lands first in the L1 queue. The job is not to solve every problem — it is to accurately classify, document, and route every alert.

Core responsibilities:
- Monitor the SIEM dashboard and alert queue
- Triage incoming alerts using defined playbooks
- Classify each alert as a true positive, false positive, or requiring escalation
- Document investigation steps and findings in the ticketing system
- Communicate clearly with L2 when escalating

The L1 analyst typically handles dozens of alerts per shift. The skill built fastest in this role is pattern recognition — learning to distinguish genuine threats from noise quickly and consistently.

## The Alert Lifecycle

```
Event occurs on a system
    System logs the event
    SIEM aggregates and correlates logs
    Detection rule fires — alert generated
    Alert enters the L1 queue
    L1 analyst claims the alert
    Investigation performed
    Classification recorded (TP / FP)
    Escalation if required
    Case closed with documentation
```

## Practical Exercise — SIEM Triage Simulation

The room includes a simplified SIEM simulation. The scenario: a bank's SOC receives an alert for a suspicious login from an unknown IP at an unusual hour.

Steps taken:
1. Open the alert and review the details — source IP, target account, timestamp, geolocation
2. Look up the source IP in a threat intelligence tool — confirmed as a known malicious IP with prior abuse reports
3. Check whether similar activity was seen from other accounts — isolated to one
4. Classify as true positive — unauthorised access attempt
5. Escalate to L2 and block the source IP via firewall rule

This exercise condenses the real workflow into an accessible format. In practice, each step involves more tooling, more context, and more uncertainty — but the logical flow is identical.

## Key Takeaways

The value of an L1 analyst is not raw technical ability — it is consistent, structured thinking applied under volume. A methodical analyst who documents clearly and escalates appropriately is more valuable than a technically skilled one who skips documentation and makes assumptions. The paper trail is how the organisation learns from incidents.
